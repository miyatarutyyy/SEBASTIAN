#+TITLE: SEBASTIAN MVP 仕様書

* 1. 目的

この仕様書は、SEBASTIAN MVP の実装対象を定義する。

MVP では、Steward が生活状態を管理し、起床シーケンスによってユーザーを =sleeping= から =awake= へ遷移させる。

企画上の背景や将来構想は =sebastian-mvp-plan.org= に記述し、本書では実装に必要な状態、イベント、画面表示、保存データ、受け入れ条件を扱う。

* 2. 用語

- SEBASTIAN
  - ユーザーの日常生活を支援するプロダクト

- Steward
  - 生活状態を管理し、必要な生活シーケンスを呼び出す上位オーケストレーター

- 起床シーケンス
  - ユーザーを =sleeping= から =awake= へ遷移させる生活シーケンス

- 起床タイマー
  - =waking= に入ってから =awake= に到達するまでの経過時間と、起床目標時間までの残り時間表示

* 3. 状態モデル

** 3.1 LifeState

MVP で扱う生活状態は次の 3 種類とする。

#+BEGIN_SRC typescript
type LifeState = "sleeping" | "waking" | "awake";
#+END_SRC

各状態の意味:

- =sleeping=
  - 睡眠中
  - =目覚めた= 操作を受け付ける

- =waking=
  - 起床シーケンス実行中
  - 起床タイマーを表示する
  - =起床完了= 操作を受け付ける

- =awake=
  - 起床完了後
  - 起床結果を表示する

** 3.2 初期状態

初回表示時に保存済み状態が存在しない場合、=life_state= は =awake= とする。

理由:

- ユーザーが普段アプリを開いた時に、睡眠中として扱われるのは不自然である
- MVP では =睡眠状態にする= 開発用操作によって =sleeping= へ遷移できる

* 4. 保存データ

MVP ではサーバー保存やデータベース保存を行わない。

ブラウザ再読み込みで状態が失われないように、次の値を =localStorage= に保存する。

=localStorage= は MVP の暫定保存先である。将来的には Steward の状態保存先をデータベースへ置き換える前提で、保存処理は実装上分離しやすい形にする。

** 4.1 保存キー

保存キーは =sebastian:steward:state= とする。

** 4.2 保存形式

#+BEGIN_SRC typescript
type StewardState = {
  lifeState: LifeState;
  wakeStartedAt: string | null;
  wakeCompletedAt: string | null;
};
#+END_SRC

日時は ISO 8601 文字列で保存する。

例:

#+BEGIN_SRC json
{
  "lifeState": "waking",
  "wakeStartedAt": "2026-07-14T21:30:00.000Z",
  "wakeCompletedAt": null
}
#+END_SRC

** 4.3 不正データ時の扱い

=localStorage= の値が存在しない、JSON として不正、または想定外の状態値を含む場合は、保存値を破棄して初期状態へ戻す。

初期状態:

#+BEGIN_SRC json
{
  "lifeState": "awake",
  "wakeStartedAt": null,
  "wakeCompletedAt": null
}
#+END_SRC

* 5. イベント仕様

** 5.1 睡眠状態にする

MVP 用の開発操作である。

実行条件:

- どの =life_state= からでも実行できる

状態変更:

- =life_state= を =sleeping= にする
- =wake_started_at= を =null= にする
- =wake_completed_at= を =null= にする

保存:

- 変更後の状態を =localStorage= に保存する

UI 表示:

- 操作名は =睡眠状態にする=
- 開発用操作であることが分かる文言を併記する

** 5.2 目覚めた

起床シーケンスを開始するユーザー操作である。

実行条件:

- =life_state= が =sleeping= の場合のみ実行できる

状態変更:

- =life_state= を =waking= にする
- =wake_started_at= に現在時刻を保存する
- =wake_completed_at= を =null= にする

保存:

- 変更後の状態を =localStorage= に保存する

UI 表示:

- 操作名は =目覚めた=

** 5.3 起床完了

起床シーケンスを完了するユーザー操作である。

実行条件:

- =life_state= が =waking= の場合のみ実行できる

状態変更:

- =life_state= を =awake= にする
- =wake_completed_at= に現在時刻を保存する
- =wake_started_at= は変更しない

保存:

- 変更後の状態を =localStorage= に保存する

UI 表示:

- 操作名は =起床完了=

* 6. 状態遷移

MVP の状態遷移は次の通り。

#+BEGIN_SRC text
awake --睡眠状態にする--> sleeping
sleeping --目覚めた--> waking
waking --起床完了--> awake
#+END_SRC

補足:

- =睡眠状態にする= は開発用操作のため、どの状態からでも =sleeping= へ遷移できる
- =waking= から =sleeping= へ戻る通常操作は実装しない
- =sleeping= から =awake= へ直接遷移する通常操作は実装しない
- 一時停止、時間切れ、自動完了は実装しない

* 7. タイマー仕様

** 7.1 表示条件

起床タイマーは =life_state= が =waking= の場合に表示する。

起床タイマーは、経過時間と残り時間の両方を扱う。ただし画面上では一度にどちらか一方を主表示する。

** 7.2 起床目標時間

MVP では起床目標時間を 30 分固定とする。

#+BEGIN_SRC typescript
const WAKE_TARGET_SECONDS = 30 * 60;
#+END_SRC

起床目標時間のカスタマイズは MVP 対象外とする。

** 7.3 経過時間の計算方法

起床タイマーは、=wake_started_at= から現在時刻までの経過時間として計算する。

1 秒ずつ値を加算するのではなく、現在時刻との差分から表示値を計算する。

理由:

- ブラウザのタブ非アクティブ化による =setInterval= の遅延に強くするため
- ページ再読み込み後も保存済み開始時刻から復元できるようにするため

** 7.4 残り時間の計算方法

残り時間は、起床目標時間から経過時間を引いて計算する。

#+BEGIN_SRC typescript
remainingSeconds = Math.max(0, WAKE_TARGET_SECONDS - elapsedSeconds);
#+END_SRC

残り時間が 0 になっても、時間切れ状態には遷移しない。

残り時間が 0 になった後も、残り時間表示は =00:00= のままとする。経過時間は =wake_started_at= からの差分として増え続ける。

** 7.5 表示形式

表示形式は =MM:SS= とする。

例:

- 0 秒: =00:00=
- 65 秒: =01:05=
- 3600 秒: =60:00=

MVP では時間単位の表示には切り替えない。

経過時間と残り時間の両方に同じ表示形式を使う。

** 7.6 タイマー表示切り替え

起床タイマーの主表示は、時計表示をタップすることで切り替えられる。

表示モード:

- =elapsed=
  - 経過時間を主表示する
  - ラベルは =目覚めてからの経過時間=

- =remaining=
  - 残り時間を主表示する
  - ラベルは =起床目標までの残り時間=

初期表示モード:

- =remaining=

補足:

- 表示モードは UI 上の一時状態であり、MVP では保存しない
- 時計表示はタップ可能な =button= として実装する
- タップ時に =elapsed= と =remaining= を交互に切り替える

** 7.7 残り時間 0 後の誘導

残り時間が =00:00= になった後も =life_state= は =waking= のままとする。

表示:

- 残り時間は =00:00= のまま表示する
- =起床完了= ボタンを押すよう誘導するメッセージを表示する

文言:

- =起床目標を過ぎています。もう起きたと判断できたら起床完了してください。=

** 7.8 起床完了後の所要時間

=wake_started_at= と =wake_completed_at= が両方存在する場合、起床にかかった時間を表示する。

所要時間は =wake_completed_at - wake_started_at= で計算する。

* 8. 時刻表示仕様

起床開始時刻と起床完了時刻は、ユーザーのローカル時刻で =HH:MM= 形式として表示する。

例:

- =07:05=
- =23:40=

日付は MVP では表示しない。

* 9. 画面仕様

** 9.1 共通表示

全状態で次を表示する。

- アプリ名: =SEBASTIAN=
- 上位管理者名: =Steward=
- 現在の生活状態
- SEBASTIAN からの短いメッセージ
- 次の行動の選択肢
- 開発用操作 =睡眠状態にする=

MVP はモバイルファーストで実装する。

UI 方針:

- 主要操作は指で押しやすいサイズの =button= として実装する
- 全操作は =button= 要素として実装する
- キーボード操作を前提にした追加仕様は MVP では定義しない

** 9.2 sleeping 表示

=life_state= が =sleeping= の場合、次を表示する。

- 状態表示: =sleeping=
- 主操作: =目覚めた=
- 補足メッセージ: 目覚めたら起床シーケンスを開始する旨

表示しないもの:

- 起床タイマー
- =起床完了= ボタン
- 起床結果

** 9.3 waking 表示

=life_state= が =waking= の場合、次を表示する。

- 状態表示: =waking=
- 起床タイマー
  - 時計表示はタップで経過時間と残り時間を切り替える
  - 初期表示は残り時間
- 起床開始時刻
- 主操作: =起床完了=
- 補足メッセージ: 二度寝しないと判断できたら完了する旨

表示しないもの:

- =目覚めた= ボタン
- 起床結果

残り時間が =00:00= になった後は、=起床完了= を押すよう誘導するメッセージを表示する。

** 9.4 awake 表示

=life_state= が =awake= の場合、次を表示する。

- 状態表示: =awake=
- メッセージ: =おはようございます=
- 次の行動の選択肢
- 起床結果
  - 起床開始時刻
  - 起床完了時刻
  - 起床にかかった時間

表示条件:

- 起床結果は =wake_started_at= と =wake_completed_at= が両方存在する場合のみ表示する

表示しないもの:

- =目覚めた= ボタン
- =起床完了= ボタン
- 実行中の起床タイマー

次の行動の選択肢:

- =睡眠状態にする=

=睡眠状態にする= は開発用操作であるため、UI 上で開発用であることを示す。

* 10. UI 文言

MVP で使用する主要文言は次の通り。

- アプリ名: =SEBASTIAN=
- 上位管理者名: =Steward=
- awake メッセージ: =おはようございます=
- 次の行動ラベル: =次の行動=
- 開発用操作: =睡眠状態にする=
- 開発用操作補足: =開発用=
- 起床開始操作: =目覚めた=
- 起床完了操作: =起床完了=
- 状態ラベル: =現在の生活状態=
- 起床経過時間ラベル: =目覚めてからの経過時間=
- 起床残り時間ラベル: =起床目標までの残り時間=
- 起床結果ラベル: =起床にかかった時間=
- 残り時間 0 後の誘導: =起床目標を過ぎています。もう起きたと判断できたら起床完了してください。=

* 11. 実装対象画面

第一リリースでは、既存画面を SEBASTIAN MVP 画面で置き換える。

要件:

- アプリのトップ画面として SEBASTIAN MVP 画面を表示する
- 既存のトップ画面 UI は第一リリースの対象外とする

* 12. エラー・例外時の扱い

** 12.1 wakeStartedAt がない waking

=life_state= が =waking= で =wake_started_at= が存在しない場合は、不正状態とみなす。

扱い:

- 初期状態へ戻す
- =localStorage= も初期状態で上書きする

** 12.2 wakeCompletedAt がない awake

=life_state= が =awake= で =wake_completed_at= が存在しない場合でも許容する。

理由:

- 初期状態が =awake= であり、起床結果が存在しないケースがあるため

この場合、起床結果は表示しない。

** 12.3 wakeCompletedAt が wakeStartedAt より前

=wake_completed_at= が =wake_started_at= より前の場合は、不正状態とみなす。

扱い:

- 初期状態へ戻す
- =localStorage= も初期状態で上書きする

* 13. テスト方針

MVP では次の観点をテストする。

対象:

- 状態遷移
- Steward 状態の保存と復元
- 不正データからの復旧
- タイマー計算
- タイマー表示切り替え

補足:

- 現時点の保存先は =localStorage= だが、将来的にデータベースへ置き換える前提とする
- テストでは保存先の具体実装に依存しすぎず、状態を読み書きする境界を分けて検証できる形を目指す

* 14. 受け入れ条件

** 14.1 初期表示

- 保存済み状態がない場合、=life_state= は =awake= として表示される
- =おはようございます= が表示される
- 次の行動の選択肢が表示される
- 開発用の =睡眠状態にする= 操作が表示される
- 起床結果は表示されない

** 14.2 睡眠状態への遷移

- =睡眠状態にする= を押すと =life_state= が =sleeping= になる
- =wake_started_at= と =wake_completed_at= はクリアされる
- =目覚めた= ボタンが表示される

** 14.3 起床シーケンス開始

- =sleeping= 状態で =目覚めた= を押すと =life_state= が =waking= になる
- =wake_started_at= が保存される
- 起床タイマーが表示される
- =起床完了= ボタンが表示される

** 14.4 起床タイマー

- =waking= 中、起床タイマーが =MM:SS= で表示される
- 起床タイマーの初期表示は残り時間である
- 時計表示をタップすると経過時間と残り時間が切り替わる
- 残り時間は 30 分からの差分として表示される
- 残り時間が 0 になっても、状態は自動変更されない
- 残り時間が 0 になった後も、残り時間表示は =00:00= のままである
- 残り時間が 0 になった後、=起床完了= を押すよう誘導するメッセージが表示される
- ページを再読み込みしても、保存済み =wake_started_at= から経過時間と残り時間が復元される

** 14.5 起床完了

- =waking= 状態で =起床完了= を押すと =life_state= が =awake= になる
- =wake_completed_at= が保存される
- 起床にかかった時間が表示される
- 起床開始時刻と起床完了時刻は =HH:MM= で表示される
- =目覚めた= と =起床完了= は表示されない

** 14.6 不正保存データ

- =localStorage= の保存値が不正な場合、初期状態へ戻る
- 画面がクラッシュしない

** 14.7 既存画面置換

- アプリのトップ画面として SEBASTIAN MVP 画面が表示される
- 既存トップ画面は表示されない

* 15. MVP 対象外

本仕様書では次を扱わない。

- 本格的な睡眠シーケンス
- 支度シーケンス
- 支度チェックリスト
- 一時停止
- 時間切れ
- 通知
- サーバー保存
- ログイン
- AI 提案
- 起床完了の自動判定
- 同日複数回実行の厳密制御
