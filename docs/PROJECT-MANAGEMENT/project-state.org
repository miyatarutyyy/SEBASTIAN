* プロジェクト概要

SEBASTIAN は、ユーザーの日常生活を支援するプロダクトである。

現在の MVP では、生活全体ではなく、最初の生活状態遷移である「起床」に絞っている。

中核概念は =Steward= である。Steward は、ユーザーの生活状態を管理し、状態に応じた生活シーケンスを呼び出す上位オーケストレーターとして位置づけられている。

現在の MVP で扱う生活状態は次の 3 つである。

#+BEGIN_SRC typescript
type LifeState = "sleeping" | "waking" | "awake";
#+END_SRC

状態遷移は次の通り。

#+BEGIN_SRC text
awake --睡眠状態にする--> sleeping
sleeping --目覚めた--> waking
waking --起床完了--> awake
#+END_SRC

=睡眠状態にする= は MVP 検証用の開発操作であり、将来的には睡眠シーケンスに置き換える想定である。

* 現在の実装の要点

** 1. Steward の状態モデル

Steward の保存状態は次の形である。

#+BEGIN_SRC typescript
type StewardState = {
  lifeState: LifeState;
  wakeStartedAt: string | null;
  wakeCompletedAt: string | null;
};
#+END_SRC

日時は ISO 8601 文字列として保存される。

実装では =Date= ではなく =@js-temporal/polyfill= の =Temporal.Instant= を使っている。

理由:

- 時刻の意味を明確にするため
- 文字列の検証を安全に行うため
- 経過時間を現在時刻との差分から計算するため

初期状態は =awake= である。

#+BEGIN_SRC typescript
{
  lifeState: "awake",
  wakeStartedAt: null,
  wakeCompletedAt: null,
}
#+END_SRC

初回表示でユーザーをいきなり睡眠中扱いしないため、この初期状態になっている。

** 2. 状態遷移は純粋関数として実装されている

=features/steward/steward.ts= には、状態遷移を行う純粋関数がある。

- =resetToSleeping()=
  - どの状態からでも =sleeping= に戻す。
  - 起床開始時刻と起床完了時刻を消す。
  - MVP 用の開発操作。

- =startWaking(state, now)=
  - =sleeping= の場合だけ =waking= に遷移する。
  - =wakeStartedAt= に =now= を保存する。
  - =sleeping= 以外では同じ state 参照を返す。

- =completeWaking(state, now)=
  - =waking= かつ =wakeStartedAt= が存在する場合だけ =awake= に遷移する。
  - =wakeCompletedAt= に =now= を保存する。
  - 完了時刻が開始時刻より前の場合は遷移しない。
  - 条件を満たさない場合は同じ state 参照を返す。

無効な操作で同じ state 参照を返す設計により、React 側では「状態が変わらなかった」ことを参照比較で判断できる。

** 3. 保存データは runtime validation してから復元する

localStorage は TypeScript の型安全性の外側にあるため、保存値は =unknown= として扱い、=validateStewardState= で検証している。

検証している主な内容:

- オブジェクトであること
- =lifeState= が ="sleeping" | "waking" | "awake"= のいずれかであること
- =wakeStartedAt= と =wakeCompletedAt= が =null= または =Temporal.Instant= として解釈できる文字列であること
- =sleeping= は開始時刻・完了時刻を持たないこと
- =waking= は開始時刻を持ち、完了時刻を持たないこと
- =awake= は時刻なし、または開始時刻と完了時刻の両方を持つこと
- 完了時刻が開始時刻より前ではないこと

不正な保存値は採用せず、初期状態へ戻す。

** 4. localStorage 境界は stewardStorage.ts に分離されている

=features/steward/stewardStorage.ts= は、保存先が localStorage であることを閉じ込める層である。

保存キー:

#+BEGIN_SRC text
sebastian:steward:state
#+END_SRC

実装上の特徴:

- =window= がない環境では初期状態を返す。
- localStorage 取得や JSON parse が失敗しても UI を壊さない。
- 不正な保存値を見つけたら、初期状態を localStorage に書き戻す。
- 保存失敗は握りつぶし、UI の動作を優先する。

これは MVP の暫定保存であり、将来的に DB 保存へ移る場合はこの境界が差し替え候補になる。

** 5. React 側では useSyncExternalStore を使っている

=features/steward/useSteward.ts= は、Steward の状態を React から使う hook である。

この hook は通常の =useState= だけではなく、=useSyncExternalStore= を使っている。

現在の設計:

- モジュール内に =clientSnapshot= を 1 つ持つ。
- 初回クライアント参照時に localStorage から復元する。
- サーバー側では localStorage を読まず、固定の server snapshot を返す。
- 状態更新時はドメイン関数を呼び、変化があった場合だけ保存と通知を行う。
- =storage= イベントを監視し、別タブで状態が変わった場合にも追従する。

hook が UI に返す主な値:

#+BEGIN_SRC typescript
{
  state,
  lifeState,
  wakeStartedAt,
  wakeCompletedAt,
  isLoaded,
  resetToSleeping,
  startWaking,
  completeWaking,
}
#+END_SRC

=isLoaded= は、SSR 初期値と localStorage 復元後の値を区別するために用意されている。ただし、現時点の画面コンポーネントでは積極的には使われていない。

** 6. 起床タイマーは保存せず、時刻差分から計算する

起床タイマーの表示ロジックは =features/waking/wakingContent.ts= にある。

起床目標時間:

#+BEGIN_SRC typescript
const WAKE_TARGET_SECONDS = 30 * 60;
#+END_SRC

タイマー値そのものは保存しない。

=wakeStartedAt= と現在時刻から経過秒数を計算し、残り秒数を派生させる。

#+BEGIN_SRC text
elapsedSeconds = now - wakeStartedAt
remainingSeconds = max(0, 30min - elapsedSeconds)
#+END_SRC

この設計により、ブラウザタブが非アクティブになって =setInterval= が遅れても、再表示時に現在時刻との差分から正しい値へ戻る。

** 7. 画面は lifeState ごとにコンポーネントを切り替える

=app/page.tsx= は client component であり、=useSteward= を使って現在の lifeState を読み取る。

表示切り替え:

- =sleeping=
  - =SleepingContent=
  - 「就寝中」
  - =目覚めた= ボタン

- =waking=
  - =WakingContent=
  - 起床タイマー
  - =起床完了= ボタン

- =awake=
  - =AwakeContent=
  - 「おはようございます」
  - 起床結果
  - 開発用の =睡眠状態にする= ボタン

各画面の共通ヘッダーは =StatusHeader= で、現在の生活状態と Steward からの短いメッセージを表示する。

** 8. WakingContent は 1 秒ごとに表示時刻を更新する

=WakingContent= は内部 state として =now= を持ち、=setInterval= で 1 秒ごとに更新する。

表示モードは次の 2 つ。

- =remaining=
  - 起床目標までの残り時間

- =elapsed=
  - 目覚めてからの経過時間

タイマー表示部分をクリックすると、残り時間表示と経過時間表示を切り替える。

30 分を超えた場合は残り時間を =00:00= に固定し、起床完了を促すメッセージを表示する。時間切れ状態そのものは持たない。

** 9. AwakeContent は起床結果を表示する

=AwakeContent= は =getWakeResultContent= を使い、起床開始時刻、起床完了時刻、起床にかかった時間を表示する。

初期状態の =awake= では起床履歴がないため、起床結果は表示されない。

=waking -> awake= の遷移で =wakeStartedAt= と =wakeCompletedAt= がそろった場合だけ、起床結果が表示される。

* 開発時の注意点

** 状態を増やすとき

=LifeState= を増やす場合は、少なくとも次を同時に確認する。

- =LifeState= type
- =LIFE_STATES= set
- =validateStewardState=
- 状態遷移関数
- =StatusHeader= のメッセージ
- =app/page.tsx= の表示分岐
- 保存データの互換性
- テスト

** 保存形式を変えるとき

localStorage の保存形式を変える場合は、古い保存値をどう扱うかを決める。

現在は schema version を持っていないため、大きな形式変更をするなら以下のどちらかを選ぶ必要がある。

- 古い保存値を破棄して初期状態に戻す
- 保存データに version を追加して migration する

** タイマーを変更するとき

タイマー表示を変更しても、タイマー値そのものを保存する設計にはしない。

保存するのは起床開始時刻と完了時刻であり、経過時間と残り時間は表示時点で計算する。

** UI で状態遷移を直接作らない

画面コンポーネントで =lifeState= を直接書き換えるような設計にしない。

UI は =useSteward= から渡された command を呼び、実際の状態遷移ルールは =steward.ts= に置く。
