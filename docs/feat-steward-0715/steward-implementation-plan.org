#+TITLE: Steward 実装方針
#+DATE: 2026-07-15

* 1. 目的

SEBASTIAN の中核となる Steward を実装する。

今回の目的は、ユーザーの生活状態を Steward が管理し、状態に応じて画面コンテンツを切り替えられるようにすることである。

最初の実装対象は、MVP 仕様で定義されている起床シーケンスである。

ユーザーは次の流れで生活状態を遷移できる。

#+BEGIN_SRC text
awake --睡眠状態にする--> sleeping
sleeping --目覚めた--> waking
waking --起床完了--> awake
#+END_SRC

* 2. 正とする仕様

実装では次の仕様を正とする。

- =docs/mvp-waking-0714/sebastian-mvp-spec.org=
- 実装先は =apps/web=
- =docs/mvp-waking-0714/code/sebastian= は参考実装として扱う
- プロトタイプをそのままコピーしない

* 4. 状態モデル

MVP で扱う生活状態は次の 3 種類とする。

#+BEGIN_SRC typescript
export type LifeState = "sleeping" | "waking" | "awake";
#+END_SRC

Steward の保存状態は次の形式にする。

#+BEGIN_SRC typescript
export type StewardState = {
  lifeState: LifeState;
  wakeStartedAt: string | null;
  wakeCompletedAt: string | null;
};
#+END_SRC

日時は =Temporal.Instant= として扱い、保存時は ISO 8601 文字列にする。

Native =Temporal= は実行環境によって利用可否が分かれるため、実装では =@js-temporal/polyfill= から import して使う。

#+BEGIN_SRC typescript
import { Temporal } from "@js-temporal/polyfill";
#+END_SRC

初期状態は次の通り。

#+BEGIN_SRC typescript
export const INITIAL_STEWARD_STATE: StewardState = {
  lifeState: "awake",
  wakeStartedAt: null,
  wakeCompletedAt: null,
};
#+END_SRC

* 5. 保存方針

保存先は MVP の暫定対応として =localStorage= を使う。

保存キー:

#+BEGIN_SRC text
sebastian:steward:state
#+END_SRC

保存処理は UI や hook に直接埋め込まず、=stewardStorage.ts= に分離する。

不正データは復元しない。

不正データの例:

- JSON として parse できない
- =lifeState= が想定外
- =waking= なのに =wakeStartedAt= がない
- =wakeCompletedAt= が開始時刻より前
- 日時文字列が =Temporal.Instant= として解釈できない

不正データを検出した場合は、初期状態へ戻す。

* 6. Steward の責務

Steward が持つ責務:

- 現在の生活状態を保持する
- 状態遷移を制御する
- 起床シーケンスの開始時刻と完了時刻を記録する
- 保存可能な状態を提供する
- UI がどの状態コンテンツを表示すべきか判断できる情報を提供する

Steward が持たない責務:

- 画面の見た目
- 起床タイマーの表示モード
- 起床結果の文言
- 起床シーケンス内の細かい UI 表示
- 将来の生活イベントの registry 管理

* 7. 関数設計

=features/steward/steward.ts= には純粋関数を置く。

実装する関数:

#+BEGIN_SRC typescript
resetToSleeping(): StewardState
startWaking(state: StewardState, now: Temporal.Instant): StewardState
completeWaking(state: StewardState, now: Temporal.Instant): StewardState
validateStewardState(value: unknown): StewardState | null
getWakeDurationSeconds(state: StewardState, now: Temporal.Instant): number | null
#+END_SRC

状態遷移のルール:

- =resetToSleeping=
  - どの状態からでも実行できる
  - =lifeState= を =sleeping= にする
  - =wakeStartedAt= と =wakeCompletedAt= を =null= にする

- =startWaking=
  - =sleeping= の場合のみ実行する
  - =lifeState= を =waking= にする
  - =wakeStartedAt= に現在時刻を入れる
  - =wakeCompletedAt= を =null= にする

- =completeWaking=
  - =waking= の場合のみ実行する
  - =lifeState= を =awake= にする
  - =wakeCompletedAt= に現在時刻を入れる
  - =wakeStartedAt= は保持する

実行条件を満たさない場合は、状態を変更しない。

* 8. React hook

=features/steward/useSteward.ts= に React hook を実装する。

責務:

- 初回マウント後に localStorage から復元する
- 状態変更後に localStorage へ保存する
- UI から呼び出す command を提供する
- SSR 中に =window= や =localStorage= へ触らない

返す値の例:

#+BEGIN_SRC typescript
{
  state,
  lifeState,
  wakeStartedAt,
  wakeCompletedAt,
  resetToSleeping,
  startWaking,
  completeWaking,
}
#+END_SRC

* 9. 起床シーケンス表示

起床タイマーの表示ロジックは Steward 本体から分離する。

=features/waking/wakingContent.ts= に起床シーケンス固有の派生値を置く。

扱う値:

- 起床目標時間 30 分
- 経過秒数
- 残り秒数
- =MM:SS= 表示
- 残り時間が 0 になったかどうか

起床目標時間:

#+BEGIN_SRC typescript
export const WAKE_TARGET_SECONDS = 30 * 60;
#+END_SRC

タイマーは保存しない。
表示時点の =Temporal.Instant= と =wakeStartedAt= の差分から計算する。

* 10. 画面構成

=app/page.tsx= は client component とし、=useSteward= を使って状態別コンテンツを切り替える。

用意するコンポーネント:

#+BEGIN_SRC text
components/
  SleepingContent.tsx
  WakingContent.tsx
  AwakeContent.tsx
#+END_SRC

画面切り替え:

#+BEGIN_SRC typescript
if (steward.lifeState === "sleeping") {
  return <SleepingContent onWake={steward.startWaking} />;
}

if (steward.lifeState === "waking") {
  return (
    <WakingContent
      wakeStartedAt={steward.wakeStartedAt}
      onComplete={steward.completeWaking}
    />
  );
}

return (
  <AwakeContent
    wakeStartedAt={steward.wakeStartedAt}
    wakeCompletedAt={steward.wakeCompletedAt}
    onSleep={steward.resetToSleeping}
  />
);
#+END_SRC

* 12. テスト方針

最低限、純粋関数はテストしやすい形で実装する。

今回の段階では、まず =pnpm lint= と =pnpm build= を通すことを完了条件にする。

追加テストを入れる場合は、優先して確認する対象は次の通り。

- 初期状態が =awake= になる
- =sleeping= から =waking= に遷移できる
- =waking= から =awake= に遷移できる
- 不正な保存データが初期状態へ戻る
- =waking= 中の経過時間が =wakeStartedAt= から復元される
